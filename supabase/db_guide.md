# LangBridge 프로젝트 데이터베이스 및 데이터 흐름 가이드

이 문서는 LangBridge 프로젝트의 주요 데이터베이스 구조와 각 페이지별 데이터 흐름을 정리한 가이드입니다.

---

## 1. 핵심 DB 구성 및 역할

### [ 콘텐츠 및 기본 정보 ]
*   **edu_videos**: 운영자가 등록한 공식 어학 강의 영상 데이터를 관리합니다.
*   **videos**: 사용자가 업로드하거나 추가한 일반 학습 영상의 메타데이터(제목, 유튜브 ID, 공개 여부 등)를 저장합니다.
*   **transcripts**: 영상의 각 문장별 원문 자막과 순서 정보를 담고 있습니다.
*   **translations**: 자막에 매칭되는 다국어(주로 한국어) 번역 데이터를 저장합니다.
*   **audio_content**: 개별 문장 학습용 오디오 파일 정보와 문장 데이터를 관리합니다.
*   **words**: 단어 원형, 품사(배열), 품사별 뜻(JSON), 성별, 변화형 및 동사 변화표(JSON) 등 상세 어휘 데이터를 관리합니다.

### [ 학습 진행 상태 ]
*   **script_progress**: 단어 배열 학습(Word Scramble) 시, 개별 문장에 대한 유저별 학습 상태(연속 정답 횟수, 마스터 여부 등)를 기록합니다.
*   **video_progress**: 특정 영상(videos) 전체에 대해 유저가 얼마나 학습했는지(진행률, 마스터한 문장 수 등)를 요약하여 보여줍니다.
*   **edu_video_progress**: 영상(edu_videos) 플레이어에서의 시청 기록(마지막 재생 위치, 누적 시청 시간 등)을 관리합니다. (이전 명칭: video_learning_progress)
*   **sp_attempts**: 단어 배열 학습 시 발생하는 모든 정답/오답 시도 이력을 상세히 기록합니다.

### [ 유저 커뮤니티 및 공유 ]
*   **board_posts**: 유저들이 학습 스크립트(CSV) 파일이나 팁을 공유하는 게시판의 데이터(제목, 본문, 첨부파일명, 조회수 등)를 저장합니다.

---

## 2. 페이지별 데이터 흐름 (Data Flow)

### ① 메인 페이지 (홈)
*   **흐름**: 서비스 전체의 핵심 콘텐츠들을 요약해서 브라우징할 수 있게 합니다.
*   **동작**:
    1.  **어학 강의**: `edu_videos`에서 운영자 추천 강의 최신 6개를 가져옵니다.
    2.  **LB 학습 영상**: `videos`에서 전체 공개(public) 설정된 최신 영상 6개를 노출합니다.
    3.  **LB 문장 학습**: `audio_content`에서 최신 문장 60개를 가져와 `lang_categories`별로 분류해 보여줍니다.
    4.  **내 학습 영상**: 로그인 유저에 한해 `videos`에서 본인이 직접 업로드한 영상 최근 10개를 가져와 보여줍니다.
        *   *참고: 전체 보기(`/my-videos`) 페이지에서는 직접 업로드한 영상뿐만 아니라, 다른 사람의 공개 영상을 내 카테고리에 '담기(저장)'한 리스트까지 모두 포함하여 보여줍니다.*

### ② 내 학습 영상 전체 페이지 (`/my-videos`)
*   **흐름**: 내가 관리하거나 학습 중인 모든 영상 콘텐츠를 체계적으로 확인합니다.
*   **동작**:
    1.  내가 직접 업로드한 영상(`uploader_id`)과 내 카테고리에 저장한 공개 영상(`user_category_videos`) 목록을 합쳐서 가져옵니다.
    2.  각 영상에 대한 나의 학습 요약 정보(`video_progress`)를 조회하여 **마스터한 문장 수와 학습 완료율(%)**을 바 형태로 표시합니다.
    3.  유저가 생성한 카테고리(`user_categories`)별로 영상을 그룹화하여 보여줍니다.

### ③ LB 학습 영상 목록 페이지 (`/lb-videos`)
*   **흐름**: 모든 공개 학습 영상을 갤러리 형태로 탐색하고, 클릭 한 번으로 내 카테고리에 담아 관리합니다.
*   **동작**:
    1.  `videos` 테이블에서 `visibility = 'public'`인 모든 영상을 통합 리스트로 가져옵니다.
    2.  로그인 유저의 경우 `user_category_videos` 정보를 일괄 조회하여, 이미 내 리스트에 담긴 영상에는 카테고리 배지를 표시합니다.
    3.  각 영상 카드에 있는 **'담기' 버튼**을 통해 실시간으로 `user_category_videos` 매핑을 추가하거나 제거할 수 있습니다.
    *   *참고: 이 페이지는 콘텐츠 탐색 및 보관 중심이므로 언어별 분류나 상세 학습 진행률은 표시하지 않습니다.*

### ④ 어학 강의 페이지 (`/videos`)
*   **흐름**: 운영자 강의 시청과 시청 기록 관리에 집중합니다.
*   **동작**:
    1.  `edu_videos` 테이블에서 전체 강의 목록을 불러오고 채널별로 필터링합니다.
    2.  `edu_video_progress`를 참조하여 내가 이 영상을 어디까지 봤는지, 총 얼마나 공부했는지 정보를 가져와 표시합니다.
    3.  영상 시청 도중 중단하거나 종료할 때 실시간 재생 위치를 `edu_video_progress`에 업데이트합니다.

### ③ 영상 상세(시청) 페이지 (`/my-videos/[id]`)
*   **흐름**: 영상 시청과 자막 학습, 개인 메모를 동시에 진행합니다.
*   **동작**:
    1.  영상 ID 기반으로 `videos`, `transcripts`, `translations`를 한꺼번에 로드하여 화면에 자막 리스트를 구성합니다.
    2.  사용자가 특정 구간에 메모를 작성하면 `user_notes`에 저장하고, 자막 리스트 옆에 표시합니다.
    3.  진입 시 `videos` 테이블의 조회수(`view_count`)를 1 증가시킵니다.

### ④ 단어 배열 학습 페이지 (`/word-scramble`)
*   **흐름**: 자막 데이터를 직접 배열하며 문장을 암기하는 인터랙티브 학습입니다.
*   **동작**:
    1.  **진입**: `initScriptProgress`를 통해 원본 자막(`transcripts`)을 유저 전용 학습 행인 `script_progress`로 복사하여 생성합니다.
    2.  **진행**: 정답 확인 시 `script_progress`의 정답 횟수와 상태(`learning` -> `mastered`)를 변경하고 `sp_attempts`에 기록을 남깁니다.
    3.  **집계**: 매 결과 제출 시마다 영상 전체의 완성도를 다시 계산하여 `video_progress` 테이블의 요약 정보를 갱신합니다.

### ⑤ 문장/오디오 학습 (`/lb-audio`)
*   **흐름**: 짧은 문장 위주의 오디오를 듣고 카테고리별로 학습합니다.
*   **동작**:
    1.  `lang_categories`에서 카테고리 목록을 가져온 후, 선택된 그룹에 속한 `audio_content`들을 리스트업합니다.
    2.  개별 오디오 재생 및 학습 현황을 해당 테이블 내에서 관리합니다.

### ⑥ 커뮤니티 게시판 페이지 (`/board`)
*   **흐름**: 사용자들이 직접 만든 CSV 학습 파일이나 노하우를 자유롭게 공유하며 소통하는 공간입니다.
*   **동작**:
    1.  `board_posts` 테이블에서 게시물들을 최신순으로 가져오며, `LIMIT`과 `OFFSET`을 이용해 페이지네이션을 처리합니다.
    2.  특정 게시글로 진입하면(`board/[id]`) 게시글 데이터를 로드하고, 조회수(`view_count`)를 1만큼 올립니다.
    3.  로그인된 유저는 글 작성(`/board/write`)을 통해 자신의 스크립트 파일과 노하우를 `board_posts`에 새로 업로드(`INSERT`)할 수 있습니다.
